# FamilyRoot workspace instructions

## Context recovery keyword

When the user sends `familyroot` or `FAMILYROOT-MEMORY` as a standalone context
recovery keyword, do the following before making project changes:

1. Read `PROJECT_MEMORY.md` completely.
2. Follow its references to the current decision documents when they are relevant to
   the requested work.
3. Inspect the current Git branch, working-tree status, latest local commit, and its
   relation to `origin/main`.
4. Confirm the recovered project phase, active constraints, and Git state to the user.
5. Wait for explicit authorization before changing project files.

Use Indonesian for the recovery confirmation unless the user requests another
language. Treat newer decisions in `PROJECT_MEMORY.md` and its referenced decision
records as overriding older roadmap notes.
